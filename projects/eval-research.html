Research

大模型评测体系研究

把「模型生成得好不好」这个主观问题，拆成可设计、可统计、可归因的评测工程。

Overview

评测的本质：把「好不好」变成可复现的工程

benchmark 的数量在爆炸，但打分的底层方法屈指可数。这套研究要做的，是把「看似无穷的评测」压成「有限的结构」——任何一个新 benchmark 出现，都能立刻归位。

基模评测 · base

评裸模型权重的通用能力（知识 / 推理 / 数学 / 代码 / 安全）。以离线 benchmark 为主，看绝对水平与相对排名。
头号命门：数据污染 + Goodhart 效应。

AI 产品评测 · product

评「模型 + prompt + 检索 + workflow + UI」端到端整体对真实用户/业务的价值。以在线 A/B + 真人为主，看业务结果。
头号命门：分布漂移 + 归因困难。

「大模型评测的类别在发散、方法在收敛。」 — 系统研究文档 · 顶层洞察
Framework

四层骨架 · 从「评什么」一路推到「什么条件下跑」

类别 · 评什么→ 方法 · 怎么评→ 概念 · 靠不靠谱→ 工程 · 什么条件跑

① 类别层　WHAT · 发散

评的是哪种本事：知识 / 推理 / 数学 / 代码 / 长文本 / 安全 / 多模态 / Agent。能力可无限细分，这一层决定了 benchmark 的数量爆炸。

② 方法层　HOW · 收敛

底层只是「谁来判（机器 / 模型 / 人）× 怎么判（相对 / 绝对）」的二维组合，实践中沉淀成 5 种高频打分法。

③ 概念层　WHY · 横切

判断「这次分数可信吗」的透镜：数据污染、Goodhart、held-out、位置/长度/自偏好 bias、可复现性。它横切所有类别和方法。

④ 工程层　正交

在什么部署条件下把评测跑出来：多版本 / 多卡 / 量化剪枝 / 蒸馏加速，核心是量化「加速比 ↔ 掉分」的权衡。

三组交叉才是真正的理解杠杆：类别×方法（有没有标准答案定总开关）· 方法×概念（每种方法继承一种命门）· 类别×概念（题的处境决定污染风险）。

Method

从主观鉴赏到可量化：让一群人打出一致的分

主观评测最大的敌人不是「谁对」，而是「换个评委分数就变」。把主观拆成可量化，靠三步。

拆维度 rubric

把「好看」正交拆成独立维度分项打分，别揉成一个总分；每个维度写清「对/错/好/坏」的判据和 badcase 锚点。

控标尺

少档位（3–5 档）降分歧；相对判断（GSB）比绝对打分一致性更高；左右对调消掉位置偏好。

量一致性 IAA

用 Krippendorff α / Cohen κ 量评委间一致性（α ≥ 0.80）。关键认知：IAA 低先怀疑 rubric 有歧义，而不是标注员差。

Dimensions

视频生成评测 · 六大一级维度体系

把一段视频拆成六个正交的一级维度，每个维度都配「拆解口径」和「badcase 锚点」，让标注同学可复现地执行。

01

画面质量

拆解　分辨率 / 细节保持、边缘伪影、色彩稳定、动态范围。

badcase　中远景五官糊成油画笔触；高对比边缘出现 JPEG 式彩色条纹。

02

动态稳定性

拆解　运动轨迹连续、帧间稳定、视差一致、无跳帧。

badcase　跑步时躯干平移双腿原地「溜冰」；静态背景挂画每隔几帧「呼吸」抖动。

03

主体一致性

拆解　跨镜外观 / 服装、颜色不漂移、数量守恒、ID 保持。

badcase　切镜后红裙变蓝 T；金毛越跑越变品种；两人对话中途冒出第三人。

04

指令遵循

拆解　核心动作、镜头景别/速度、风格场景、文字渲染。

badcase　要求「猫踩琴键」实际走地板；文字「HELLO」渲染成「HLEOL」。

05

音画同步

拆解　口型对齐、动作-音效卡点、声源-音色匹配、环境音对应。

badcase　口型比语音延迟约 0.5 秒；画面弹吉他却传出钢琴声。

06

整体观感

拆解　审美 / 视听语言、情绪张力、完成度、综合可用性。

badcase　六个维度单看都及格，合起来却像 PPT 翻页——没有镜头调度与情绪。

隐藏横切维度

物理合理性（重力 / 接触 / 介质交互 / 镜面反射）与 世界知识·隐含常识（钟表指针方向、QWERTY 键盘、日历日期、红绿灯顺序、手指数量）——这些不写在 prompt 里，但结果必须符合，是拉开专业度的追问点。对应学术 benchmark：VBench · PhyGenBench · VBench-2.0-Physics。
