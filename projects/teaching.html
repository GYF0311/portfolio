Positioning

这门课教什么

面向想真正落地的 AI 训练师 / 数据运营。不是讲某个平台的按钮，而是给一套可迁移的方法： 接到任何图片描述、标注、评测项目，先问人怎么做，再判断 AI 能不能做， 再把人的 SOP 翻译成 Prompt 和 Workflow，最后用验证结果证明它是否真的提效。

教什么

把人工标注规则翻译成模型可执行的 Prompt，串成能批量产初稿的 Workflow， 再用 AI Coding、Skill、API 批处理把链路做稳、做快。

给谁教

不需要会写代码。目标是会用、会调、会验收——报错不是失败，是下一步要问 AI 的材料。

学完能落地

一包图片能自动跑成 JSON / Excel / 标注平台预填数据，人工只做复核； 成果还能沉淀成作品页，面试用。

Main Line

课程主线骨架

整门课贯穿一条可交付主线。学生要记住的不是某个节点，而是这条线本身。

接需求 → 写规则 → Prompt → Workflow → 复核回流
01 · 接需求

先问人怎么标

谁用这批数据、终局长什么样、现在人怎么标、规则文档在哪、有没有已标样本、 人工效率基线、哪些错误不能接受。问清楚再动手。

02 · 写规则

把 SOP 翻译成规则文档

画面形式、主体类型、动作状态、构图、光影、色彩、风格逐字段定标签与范围； 哪些固定、哪些开放；不确定就输出 unknown 的边界兜底。

03 · Prompt

写生产级 Prompt 并迭代

用 RECIPE 框架把规则写成模型能执行的说明书，拿 3–10 张图试跑； 反复优化仍出错的字段，记为模型边界，交人工重点复核。

04 · Workflow

把 Prompt 串成流水线

单个 Prompt 只描述一张图；Workflow 解决一批图的审核、分类、分场景描述、 结构化输出。低代码（Coze）和 AI Coding 脚本是同一逻辑的两种实现。

05 · 复核回流

人工复核，问题回流

AI 出初稿、人工复核修改；审核误判、字段不稳、覆盖不全等问题回流到 规则、Prompt、few-shot 和节点，用人工修改时间下降证明真提效。

终局图

一包图片 → 可交付数据

图片审核 → 类型分类器 → 风景 / 商品 / 人物 / 动物 / 通用专项 Prompt → JSON 结果 → 人工复核入表。先讲终局，再拆过程。

Methodology

怎么教（且曼体系）

方法论决定这门课不是知识点罗列，而是学生真能上手的东西。

01

Why 讲透，但只当引子

开场先把「重要、酷、专业」立起来镇住学生，点燃好奇后利落切正题—— 不是要占多大比例，讲太长反而适得其反。

02

抓大放小，不当搬运工

吸收 → 消化 → 吐出。只讲自己真懂、能落地的点，其余砍掉， 绝不把素材原样复述一遍。

03

学十教一，教一备十

每个点先用学生视角列十个刁难问题自测，自己答不上就不讲。

04

老师做结构

把散素材拆成清晰三段再塞知识；专业名词先解释再用； 适时 call back 前面讲过的概念。

05

实操课：先甩结果，再拆原理

先甩出炸裂的成品结果 → 锚定目标并全程回扣 → 带练复现 → 最后才讲原理。 课前一定预跑出可直接复用的稳定版本，1v1 / 远程也要让学生亲手跑通最小闭环， 不让课堂变成全程旁观或现场 debug。

两节课的模块概览

课上用「图片描述」一条线讲到底；作业换成 ASR 语音评测，逼学生迁移而不是复刻。

概念阶梯
# 一层套一层
Prompt   这一次怎么做
Workflow 多步骤怎么串
Skill    同类任务以后都这么做
RECIPE 六要素

角色 · 目标 · 输入 · 规则 · 输出 · 边界。边界不是和规则打架， 是规则里的「不要做什么」——不猜身份、不编作品名、不确定就 unknown。

一样的事做两遍

同一个项目先用 web coding 脚本做一遍，再在扣子平台搭一遍， 让学生看到两种实现、同一套逻辑。

真正的门槛不是学会某个平台的按钮，而是能把人的 SOP 拆成 Prompt 和 Workflow—— 我把这条线走通了，也把它讲成了学生能亲手跑通的课。 — 高一帆 · 且曼学院 AI 训练师提效实战课
