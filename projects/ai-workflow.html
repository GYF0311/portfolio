AI Workflow

我的 AI 工作系统

一套把 AI 融进日常工作的系统：个人知识库沉淀经验，双 Agent 分工协作，小工具把重复活自动化。

Overview

三件套：知识库 + 双 Agent + 小工具

我不是「偶尔用一下 AI」，而是把它做成了一条常驻的工作流水线。原始资料被增量编译成可检索的长期知识库；两个 AI Agent 分主控与第二工程师协作干活；一批小工具把爬取、抽帧、批处理这类重复劳动交给脚本。三层各司其职，又互相反哺。

原始资料 → lorekit ingest → 知识库 wiki → query / 反哺工作 ↺ fileback 结论
01 · 知识库

corpus × lorekit

把散落的资料、录音、复盘增量编译成一座长期维护的 wiki，随时可检索、可回读、可引用。

02 · 双 Agent

Claude × Codex

主控负责规划与终审，第二工程师负责执行；通过委托链路分工，共享同一套记忆。

03 · 小工具

提效脚本集

爬虫、音视频分离抽帧、画面自动描述、批处理、训练师 Workflow 链路——重复活自动化。

Knowledge Base

corpus × lorekit：把资料编译成长期 wiki

核心思路是「增量编译」：资料不是堆进文件夹就完事，而是像代码一样被 ingest、建立反链、定期 lint 校验，逐步长成一座结构化、可检索的个人 wiki。原始资料只读、不改；分析和结论走独立的写回通道，事实源始终唯一。

ingest 摄入 → 建反链 / 索引 → query 检索 → fileback 写回 → lint 体检

ingest · 摄入

URL、文件、粘贴文本统一走摄入入口：抓取正文、归档来源、生成 wiki 页并挂上反链，原始资料落入只读区。

query · 检索

先读目录索引再命中 canonical 页面，最终答案回读知识库正文，派生草稿不当事实源——保证引用可靠。

fileback · 写回

对话里得到的结论，经确认后走受控通道写回知识库，而不是手改 canonical 页；源优先，避免双事实源。

lint · 体检

定期检查断链、孤岛节点、frontmatter 规范，保证 wiki 结构长期健康；只查结构，事实层靠人工 review 兜底。

# 一条命令摄入，一条命令检索——像管代码一样管知识
lorekit ingest "<url | file | text>"   # 抓取→归档→建页→反链
lorekit query  "视频生成评测维度"       # 先索引后正文，回读 canonical
lorekit lint                          # 断链 / 孤岛 / frontmatter 体检
Dual-Agent

Claude × Codex：主控 + 第二工程师

我把两个 AI 当成一个小团队：Claude Code 是主控，负责澄清需求、拆解任务、路由和终审；Codex 是独立的第二工程师，负责执行——爬取、清洗、批量转换、课件制作都交给它。主控通过一条委托链路把任务派出去，两边共享同一套记忆，同一个问题不用讲第二遍。

主控

Claude Code

和我对话的唯一入口。负责把模糊需求问清、拆成可执行的任务包、决定谁来做、最后审收成果。规划与判断留在主控手里。

澄清拆解 任务路由 交叉 review
第二工程师

Codex

独立长线程 Agent，接主控派来的任务包和边界，执行后交回结构化回执。可按需并发多个子 Agent 处理独立同构的子任务。

批量执行 多 Agent 并发 结构化回执
环节主控 Claude第二工程师 Codex
需求澄清和我对话，问清目标与边界—
任务拆解拆成带边界的任务包—
执行轻量琐碎自己做爬取 / 清洗 / 批量转换 / 课件
并发调度并行子 Agent可 fan-out 多个子 Agent
质量审对方成果（不自审自过）自查后交回执，供主控复核
记忆共享同一套持久记忆：习惯、路径、流程、踩过的坑、决策，两边自动加载

记忆分三层，只共享该共享的：稳定的协作规矩两边同内容加载；持久事实沉淀成共享记忆库、Agent 自治维护、我不参与；各自的私有召回则互不干扰。任务能拆就默认并发——两边的轻量子 Agent 等价并发，成本靠执行档位控制。

# 主控把任务派给第二工程师，带上工作目录与边界
dispatch --cwd <项目根> --effort high "<任务包 + 交付边界>"
  → Codex 长线程执行，自查后交回 STATUS / ARTIFACTS / NEXT 回执
  → 主控 review 通过 → 我验收
Tools

AI 提效小工具集

系统之外，我攒了一批把重复劳动自动化的小工具。它们不追求通用，只解决我手头真实反复出现的活儿——从素材采集，到多模态内容处理，再到训练师课程的全链路提效。会用 AI 做项目提效，落到实处就是这些脚本。

采集

爬虫脚本

按需抓取公开网页与图文素材，正文、图片一并落地，喂给下游知识库或分析流程。

多模态

音视频分离 / 抽帧

把视频拆成音轨与关键帧，为转写、评测、标注等下游任务准备干净的输入素材。

多模态

抽帧画面自动描述

抽出的画面交给多模态模型批量生成结构化描述，把「看图」这一步也自动化。

批处理

批量转换脚本

格式转换、重命名、逐字稿归档、命名规范化等重复动作打包成批处理，一次跑完一批。

链路

训练师 Workflow 链路

接需求 → 写规则 → Prompt → Workflow → 复核回流，把 AI 训练师的提效流程串成可复用链路。

链路

录音 → 讲义链路

录音拉逐字稿、保留说话人与时间戳、整理成结构化讲义，课程与复盘素材一条龙。

# 典型链路：一段视频 → 关键帧 → 自动描述 → 结构化产物
ffmpeg -i input.mp4 -vf "fps=1" frames/%04d.jpg   # 抽帧
describe frames/ --model multimodal > captions.jsonl  # 画面自动描述
batch captions.jsonl --into 知识库              # 批处理入库
